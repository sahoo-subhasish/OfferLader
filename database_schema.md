# Firebase Firestore Database Schema

This document outlines the active Firestore Collections and Document structures for the Algo2Offer project.

## Collections

### 1. `users`
This collection holds all data specific to a user, including authentication metadata, profile details, role, and progress tracking.

**Document ID**: Firebase Authentication UID (`user.uid`)

**Fields:**
- `fullName` (String): e.g., "John Doe"
- `email` (String): e.g., "john@example.com"
- `githubId` (String): e.g., "johndoe"
- `linkedinId` (String): e.g., "johndoe"
- `vjudgeId` (String): e.g., "johndoe"
- `leetcodeId` (String) [Optional]
- `codeforcesId` (String) [Optional]
- `role` (String): `"admin"` | `"user"`
- `createdAt` (String): ISO Date String
- `solvedProblems` (Map): A dictionary tracking the solve status of problems using the problem ID.
  - Key `[String]`: The unique problem ID (e.g., `"t2-311"`).
  - Value `[Boolean]`: The boolean status indicating if it is solved (e.g., `true`).

---

### 2. `contests`
A repository of coding contests added via the Admin Dashboard.

**Document ID**: Auto-generated by Firestore

**Fields:**
- `title` (String): The title of the contest (e.g., "Weekly Contest 400").
- `link` (String): The external URL to the contest.
- `createdAt` (Number): Unix timestamp of creation.

---

### 3. `blogs`
A repository of technical and platform blogs written using Markdown from the Admin Dashboard.

**Document ID**: Auto-generated by Firestore

**Fields:**
- `title` (String): The title of the blog post.
- `content` (String): The Markdown content of the blog post.
- `authorName` (String): Name of the admin who authored the blog.
- `createdAt` (Number): Unix timestamp of creation.
- `status` (String): Indicates if the blog is `'published'` or `'draft'`.
- `likes` (Array of Strings): UIDs of users who liked this blog.

#### Subcollection: `blogs/{blogId}/comments`
Holds comments submitted by users for a specific blog.

**Document ID**: Auto-generated by Firestore

**Fields:**
- `uid` (String): Firebase UID of the commenter.
- `authorName` (String): Display name or full name of the commenter.
- `text` (String): The comment body.
- `createdAt` (Number): Unix timestamp of creation.

---

### 4. `questions` (Proposed Future)
A global repository of problems so they can be managed via an Admin Dashboard instead of hardcoded JSON files.

**Document ID**: Auto-generated or custom specific short ID (e.g., `q_two_sum`)

**Fields:**
- `title` (String): e.g., "Two Sum"
- `topic` (String): e.g., "Arrays"
- `difficulty` (String): "Easy" | "Medium" | "Hard"
- `link` (String): Url to Leetcode/Platform
- `tier` (String): Which tier they belong to (e.g., "tier1", "basic", "master")
